313 /* ------------------------------------------------------------------------- */
314 //! FUNCTION: LTC6950_set_frf
315 //! Calculates the integer (N) and output divider (OD) SPI values
316 //! using self created 64bit math functions. For Out0 only. Use menu 3 to set dividers to
317 //! achieve desired frequencies on other outputs.
318 //!
319 //! Datasheet equations
320 //! - fvco = fpfd*(N + F)
321 //! - frf = fvco/O
322 //! - fpfd = fref/R
323 //!
324 //! can be modified to the following equations
325 //! - N = (int) (fvco/fpfd) = (int) frf*O*R/fref
326 //!
327 //! where
328 //! - N = ndiv, O= M0div in the code below
329 //!
330 //! Linduino One (Arduino Uno) is limited to 32 bit floats/double/long.
331 //! 32 bit math functions will create rounding errors with the above equations,
332 //! tha can result in frequency errors.
333 //! Therefore, the following code uses self created 64bit functions for 64bit integer math.
334 //!
335 //! - frf (33 bits) LTC6950 max frf/fvco = 1.4GHZ, which is 23 bit number (2^31 = 2.147G)
336 //! - fref (23 bits) LTC6950 min fref = 2MHz, which is a 20 bit number (2^20 = 1.05M)
337 //! - O (6 bits)
338 //! - R (10 bits)
339 //!
340 //! step 1: create 64 bit frf and fref numbers
341 //!
342 //! step 2: calculate O (output divider)
343 //!
344 //! step 3: get current R-divider setting
345 //!
346 //! step 4: calculate frf*O*R
347 //! - max bit count/resolution: 31b+6b+10b= 47b
348 //!
349 //! step 5: calculate N(11b), using value from step 1
350 //! - N = (int) frf*O*R/fref
351 //! - max bit count/resolution: 47b-20b = 27b
352 //!
353 //! step 6: find N for closest frequency - accounts for rounding with integer math
354 //! - 64b-47b=17b (using 16b instead of 17b)
355 //!
356 //! @return void
357 void LTC6950_set_frf();
358
